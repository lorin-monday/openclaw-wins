# Contributing

## Add a new win

1. Copy the template from `examples/wins/_template.md`.
2. Fill in the frontmatter.
3. Keep the narrative short and operational.
4. Run validation.
5. Open a PR.

## Review criteria

Maintainers should check:
- Is the problem clear?
- Is the solution reusable?
- Is the environment explicit?
- Are failure bounds documented?
- Is there enough evidence to trust it?

## Curation guidelines

Prefer records that:
- save future debugging time
- capture non-obvious workarounds
- document provider / environment quirks
- can be tested again later
